Always write tests before implementation files when the user has agreed on a plan that includes tests.

**Why:** User explicitly called this out when implementation files were written before tests despite the plan stating "write tests first."

**How to apply:** In any planned feature or tool build, write all test files (bats, unit, integration, fixtures, helpers) before writing the implementation files. Follow the agreed plan order strictly.
